# ⚙️ 核心系統解析 (System Mechanics)

## 1. 🕹️ 極致精準動態搖桿 (High-Precision Floating Joystick)

為了解決手遊端操控不夠精準的痛點，本作採用了「嚴謹級」操控模型：

- **全局座標運算**: 使用 `global_position` 進行向量計算，徹底排除 UI 容器與螢幕縮放誤差。
- **瞬時響應**: 移除輸入端的 `lerp` 平滑處理，提供 1:1 的零延遲轉向回饋。
- **動態中心**: `base` 會在點擊瞬間精準成形於手指位置，確保每次操作的起始向量均為 `(0,0)`。
- **死區判定**: 設定 `8.0` 像素精細死區，過濾掉手指細微顫動。

## 2. 👿 十殿閻君生成器 (Boss Engine)

- **觸發時機**: `Time % 60 == 50`。
- **生成邏輯**:
  - 從 Boss 註冊表中隨機挑選一位領主。
  - 血量加成: `BaseHP * Difficulty * 15`。
  - 大小加成: `Scale 3.0`。
  - **必掉寶物**: 擊敗後 100% 掉落 **黃金經驗珍珠**。

## 3. 🌀 無限混沌模式 (Chaos Mode)

當遊戲時間超過 20 分鐘後啟動：

- **隨機地貌**: 每次生成的怪獸波次會隨機選擇一種 `Texture2D` 背景（S1 到 S10 的所有紋理）。
- **數值膨脹**: 難度係數隨時間呈指數級增長。

## 4. 📈 難度演算 (Difficulty Scaling)

- **基礎倍率**: `1.0 + pow(Stage, 2.3)`。
- **怪物血量**: `BaseHP * Multiplier` (初始化時注入)。
- **經驗衰減**: 隨著階段提升，升級所需的經驗會線性增加，確保後期的成長壓力。
