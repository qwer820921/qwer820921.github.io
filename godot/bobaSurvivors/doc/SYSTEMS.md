# ⚙️ 核心系統解析 (System Mechanics)

## 1. 🕹️ 動態漂浮搖桿 (Floating Joystick)

為了解決傳統固定式搖桿在小螢幕上的誤觸問題，本作實作了動態校準邏輯：

- **觸發區域**: 螢幕左半部全域。
- **動態中心**: `base` 座標會在 `_gui_input` 的 `InputEventScreenTouch` 時更新。
- **死區判定**: 設定 `10.0` 像素死區，防止靜止時手抖造成的位移。
- **向量平滑**: 使用 `lerp` 處理輸出向量，讓角色移動與轉向具有「慣性」感，手感更滑順。

## 2. 👿 十殿閻君生成器 (Boss Engine)

- **觸發時機**: `Time % 60 == 50`。
- **生成邏輯**:
  - 從 Boss 註冊表中隨機挑選一位領主。
  - 血量加成: `BaseHP * Difficulty * 15`。
  - 大小加成: `Scale 3.0`。
  - **必掉寶物**: 擊敗後 100% 掉落 **黃金經驗珍珠**。

## 3. 🌀 無限混沌模式 (Chaos Mode)

當遊戲時間超過 20 分鐘後啟動：

- **隨機地貌**: 每次生成的怪獸波次會隨機選擇一種 `Texture2D` 背景（S1 到 S10 的所有紋理）。
- **數值膨脹**: 難度係數隨時間呈指數級增長。

## 4. 📈 難度演算 (Difficulty Scaling)

- **基礎倍率**: `1.0 + pow(Stage, 2.3)`。
- **怪物血量**: `BaseHP * Multiplier` (初始化時注入)。
- **經驗衰減**: 隨著階段提升，升級所需的經驗會線性增加，確保後期的成長壓力。
