# Implementation Plan: Ten-Minute Email (10分鐘信箱)

本文件詳實紀錄了「10分鐘信箱」功能的開發規格與最終實作成果。

## 1. 系統架構概要

本工具採用 **Proxy 代理模式**，由前端呼叫 Cloudflare Worker，Worker 代為與 Mail.tm API 溝通。

- **前端框架**：Next.js + Bootstrap 5 + Zustand
- **樣式方案**：CSS Modules (Light Glassmorphism)
- **後端代理**：Cloudflare Workers + Upstash Redis

---

## 2. 佈局與 RWD (最終版本)

為了提升大螢幕的資訊密度，採用了 **6:6 橫向併排佈局**。

- **左側欄 (6/12)**：臨時地址展示、延長/更換按鈕、倒數計時器。具備 `sticky` 效果。
- **右側欄 (6/12)**：收件匣清單。
- **行動端**：自動切換為 12/12 垂直堆疊。

---

## 3. UI 視覺規範 - 明亮磨砂玻璃 (Light Glassmorphism)

放棄了沉重的深色調，改用更具現代感且通透的明亮介面。

### 設計方針

- **明亮磨砂**：使用 `rgba(255, 255, 255, 0.75)` 背景搭配 `30px` 模糊。
- **高導讀性**：文字採用深石板藍 (`#1e293b`)，連結使用標準藍 (`#2563eb`)。
- **去 important 化**：透過 `.modalDialog` 高權重選擇器精準覆蓋 Bootstrap，代碼更潔淨。

---

## 4. 彈窗 UX 優化

- **標題與底部固定**：透過 Flexbox (`flex-direction: column`) 確保彈窗不超出螢幕。
- **內容內置捲動**：信件長度超過時，僅在內文區域顯示捲動條，不影響整體頁面。
- **資訊標籤化**：取消冗餘圖示，改用簡潔的對稱卡片展示發件人與時間。
